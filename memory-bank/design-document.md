# 产品设计文档 - 迹录 StepMap

## 1. 文档信息

- 产品名称：`迹录 (StepMap)`
- 文档类型：产品设计文档（Product Design Document）
- 对应需求来源：已并入当前文档（`PRD.md` 已移除）
- 当前版本：`V1.0（含 V1.1 预留）`
- 更新时间：`2026-04-25`

## 2. 设计目标

- 把旅行记录流程压缩到最少操作：打开页面、添加一条足迹、完成保存。
- 确保地图与时间线共同构成核心体验：地图负责空间记忆，时间线负责时间记忆。
- 在 V1.0 保持极简：聚焦记录与回顾，不引入社交和复杂编辑能力。
- 为 V1.1 的“旅行电影”预留信息结构和任务状态能力，不阻塞 MVP 上线。

## 3. 设计原则

- 极简优先：每个页面只保留一个主任务。
- 一致性：操作按钮、表单反馈、错误提示文案风格统一。
- 可恢复：上传失败、定位失败、网络中断时，优先保留用户输入。
- 可感知性能：关键操作要有即时反馈（loading、进度、成功提示）。
- 移动端优先：核心链路在手机竖屏下单手可完成。

## 4. 用户与场景

### 4.1 目标用户

- 纯粹记录者：不希望被社交干扰，只想留下一次到访痕迹。
- 回忆驱动者：希望通过地点和照片快速唤醒记忆。
- 轻创作者：需要一个低门槛产出旅行回顾素材的工具。

### 4.2 核心场景

- 出行中：随手拍照并立即记录位置和一句话。
- 出行后：批量补录关键地点，形成一条时间线。
- 回顾时：从地图点位进入历史旅行，快速浏览内容。

## 5. 信息架构

```text
未登录
└── 登录/注册

已登录
├── 首页（地图总览 + 旅行列表）
│   ├── 创建旅行
│   └── 进入旅行详情
└── 旅行详情
    ├── 时间线列表
    ├── 添加足迹
    └── 生成旅行电影（V1.1）
```

## 6. 页面与交互设计

### 6.1 登录/注册页

- 目标：让用户快速进入产品，不增加无关字段。
- 必备元素：
  - 邮箱输入框
  - 密码输入框
  - 登录/注册切换入口
  - 提交按钮
- 交互规则：
  - 提交时禁用按钮，避免重复请求。
  - 失败提示在字段附近显示，避免仅 toast。

### 6.2 首页（地图总览）

- 目标：让用户第一眼看到“我去过哪里”。
- 页面分区：
  - 顶部：产品名 + 新建旅行按钮 + 用户菜单
  - 中部：世界地图（足迹点）
  - 底部/侧栏：旅行列表（名称、日期、足迹数量）
- 关键交互：
  - 点击地图点位跳转对应旅行详情。
  - 点击“新建旅行”弹出创建表单或进入创建页。
  - 地图加载失败时切换列表模式并提供重试。

### 6.3 创建旅行

- 输入项：
  - 旅行名称（必填）
  - 起止日期（必填）
  - 封面图（可选）
- 校验规则：
  - 名称长度建议 1-40 字
  - 结束日期不可早于开始日期
- 反馈：
  - 成功后自动跳转旅行详情
  - 失败后保留已输入内容

### 6.4 旅行详情页（时间线）

- 目标：低认知成本浏览本次旅行。
- 页面结构：
  - 头部：旅行名称、日期区间、添加足迹按钮
  - 主体：时间线卡片（倒序）
  - 辅助：地图缩略入口（可选）
- 足迹卡片字段：
  - 照片
  - 一句话描述
  - 日期时间
  - 地点名称
- 空状态：
  - 文案示例：`还没有足迹，去添加第一条记录吧。`

### 6.5 添加足迹页/弹窗

- 输入项：
  - 照片上传（必填）
  - 地理位置（自动定位或手动选点，必填）
  - 一句话描述（必填，100 字以内）
- 关键交互：
  - 自动定位失败时，自动切换手动选点。
  - 上传失败时不清空文字输入。
  - 保存成功后返回详情页并定位到新卡片。

### 6.6 旅行电影（V1.1）

- 入口：旅行详情页按钮 `生成电影`
- 状态流转：排队中 -> 处理中 -> 已完成 / 失败
- 完成态：显示下载按钮和更新时间
- 失败态：显示重试按钮和简要原因提示

## 7. 关键流程说明

### 7.1 创建旅行流程

1. 用户点击新建旅行。
2. 填写表单并提交。
3. 服务端创建成功后返回旅行 ID。
4. 前端跳转到旅行详情并显示空时间线引导。

### 7.2 添加足迹流程

1. 用户上传照片并输入一句话。
2. 定位成功则自动填入坐标；失败则手动选点。
3. 提交后服务端保存数据并返回足迹实体。
4. 前端更新时间线和地图点位。

### 7.3 回顾浏览流程

1. 用户从首页地图或列表进入旅行详情。
2. 按倒序浏览足迹。
3. 若需要，返回首页继续查看其他旅行。

## 8. 状态与反馈设计

### 8.1 Loading 状态

- 页面首次加载：骨架屏或统一 loading 组件。
- 提交按钮：提交中禁用并显示进度文案（如“保存中...”）。

### 8.2 Empty 状态

- 无旅行：首页展示创建引导。
- 无足迹：旅行详情展示添加引导。

### 8.3 Error 状态

- 网络错误：通用重试提示 + 保留输入。
- 定位错误：提供手动选点替代路径。
- 上传错误：支持重传，避免用户重新填写全部字段。

## 10. 非功能设计要求

- 性能目标：
  - 足迹保存 <= 2 秒
  - 首页地图首次可交互 <= 3 秒（4G 网络）
- 安全目标：
  - 密码加盐哈希
  - 所有业务接口鉴权
  - 图片访问签名 URL
- 隐私目标：
  - 默认私有，不提供公开广场
  - 删除账户可级联删除个人数据

## 11. 指标与埋点建议

- 激活指标：注册后 24 小时内创建首个旅行的比例
- 核心行为指标：每位用户每周新增足迹数
- 留存指标：7 日回访率
- 质量指标：
  - 足迹保存成功率
  - 图片上传失败率
  - 地图加载失败率

建议埋点事件：

- `sign_up_success`
- `trip_created`
- `footprint_created`
- `map_pin_clicked`
- `movie_generation_requested`（V1.1）

## 12. 可用性与文案规范

- 文案短句化，优先动词开头：`创建旅行`、`添加足迹`、`重试`
- 错误提示可操作：不仅告知失败，还给下一步动作
- 表单字段出现错误时，直接定位到对应字段并高亮

## 13. 版本规划

### V1.0（MVP）

- 用户系统
- 创建旅行
- 添加足迹
- 旅行时间线
- 地图总览

### V1.1

- 旅行电影异步生成
- 任务状态查询与失败重试

### V2（探索）

- 同地点多次旅行对比
- 只读分享链接
- AI 扩写旅行短文

## 14. 风险与应对

- 风险：地图服务不稳定影响核心体验  
  应对：增加列表降级视图和重试机制

- 风险：移动端上传失败率高  
  应对：压缩策略、断点续传预案、失败后保留表单

- 风险：V1.1 视频生成耗时或失败率高  
  应对：异步队列、任务重试、先支持小规模图片数量
